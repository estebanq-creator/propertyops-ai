### Long Form (FAQ / Investor Deck Appendix)

**Q: Why isn't Tony shipping with Laura?**

**A:** Disciplined sequencing. Laura establishes trust, compliance posture, and agent accuracy before expanding to tenant-facing operations. Tony has higher automation bias risk and habitability liability — we validate the foundation first.

**Q: What does "draft-only" mean?**

**A:** Tenants cannot send messages autonomously. All Tony-drafted messages appear in CEO Mission Control for one-click approve/reject. Only approved messages are visible to landlords. This prevents miscommunication and establishes accuracy baseline.

**Q: When does Tony become autonomous?**

**A:** Phase 4 (Controlled Autonomy) — after:
- Laura pilot completes with <5% false positive rate
- 50/50 review gate bypassed
- Tony beta validates message accuracy (3+ months)
- Compliance audit passed (zero Fair Housing violations)
- Historical data shows safe automation patterns

**Q: What happens if Tony misses a habitability issue?**

**A:** Three safeguards:
1. Habitability classification triggers RED flag → CEO immediate review
2. All tenant messages CEO-approved in beta
3. Audit trail: CEO userId + timestamp on every approval

Liability remains with human approver (CEO), not the agent. This is intentional.

**Q: Is Tony a chatbot?**

**A:** No. Tony is a drafting agent that prepares messages for CEO approval. Tenants do not chat with Tony. Tony drafts responses based on maintenance request context, landlord preferences, and historical patterns — then waits for approval.

**Q: What's the endgame for Tony?**

**A:** Controlled autonomy for routine maintenance (e.g., "When is trash pickup?") with human escalation for edge cases (habitability, legal, financial). Think "exception-only review" — 95% autonomous, 5% human. But we earn that through validation, not announce it on day one.

## Messaging Don'ts

### Never Say

| ❌ Don't Say | ✅ Say Instead |
|-------------|----------------|
| "Laura screens tenants" | "Laura analyzes application documents" |
| "Laura approves/rejects applicants" | "Laura flags document anomalies for your review" |
| "Laura gives you a score" | "Laura provides evidence citations" |
| "Tony talks to tenants" | "Tony drafts messages for your approval" |
| "Tony is autonomous" | "Tony is draft-only in beta" |
| "Automated tenant communication" | "Human-approved tenant messaging" |
| "AI decides who gets approved" | "AI flags anomalies, you decide" |
| "Billing is automated" | "Manual invoicing for pilots" |
| "General availability" | "Pilot" or "Closed beta" |
| "Phase 2 launches next week" | "Phase 2 follows Laura validation" |
