# CEO Agent - Metrics

Monitor the business with metrics that support executive decisions.

## Phase 0 KPIs (Weeks 1–3)

**Primary KPI:** ≥40% reduction in per-application review time within 30 days of active use (measured per account against a Week 0 baseline captured before platform access).

**Secondary KPI:** Document anomaly catch rate — log every Laura flag, label each as genuine / false positive / inconclusive at pilot conclusion. Establishes the precision baseline used in Account 2 sales conversations.

Additional gates:
- Paying customers secured: target 5
- Willingness to pay confirmed: Yes/No
- Document Integrity Report format validated: Yes/No
- Manual analysis turnaround time: < 24 hrs
- ≥2 of 5 accounts generate a qualified referral (Referral Engine)

Exit: 5 accounts confirmed, KPI baselines captured, Day 35 case study drafted.

See `agents/ceo/PHASE0-PROTOCOLS.md` for full measurement protocol, checkpoint schedule, and Referral Engine triggers.

## Phase 1 KPIs (Weeks 4–10)

- Secure intake pipeline live: Yes/No
- PAM controls implemented: Yes/No
- PII redaction pre-analysis: Yes/No
- Laura anomaly detection accuracy: > 85%
- Phase 0 customer retention: ≥ 80%
- New customers onboarded: 10–20

Exit: Laura autonomous, 10+ active accounts, PAM audit passed.

## Phase 2 KPIs (Months 3–6)

- Mission Control dashboard live: Yes/No
- Tony draft acceptance rate (no edits): > 70%
- Average landlord time-in-app per day: > 10 min
- Tenant messages processed per week (per account): > 5
- Work orders drafted per week (per account): > 2
- Net Promoter Score: > 40

Exit: 60+ days of approval data per account, vendor preference data captured, habit loop confirmed via retention metrics.

## Phase 3 KPIs (Months 6–9)

- Auto-approval threshold logic defined: Yes/No
- First autonomous dispatches live: Yes/No
- False-positive rate on autonomous actions: < 2%
- Customer opt-in to autonomy features: > 50%

Exit: Seed round close, autonomous actions live with override audit trail.

## Operational Metrics (Ongoing)

- response time
- time to resolution
- first-time fix rate
- cost per repair
- vendor callback rate
- overdue work-order count
- tenant retention
- delinquency trend
- documentation completeness

## Use

- use metrics to spot drift, not just summarize history
- compare current performance to trend, not isolated anecdotes
- escalate when metrics worsen repeatedly without a clear cause

## Decision Lens

For any recommendation, ask:
- which metric improves?
- which metric worsens?
- what is the tradeoff?
- which phase does this metric belong to, and are we in that phase?
